
# 基于 ARM-GCC 的 STM32 调试方法

对于 STM32 系列芯片的开发，虽然市场有KEIL、IAR等集成开发环境可供使用，但考虑到若项目有一定的跨平台性，则上述的IED基本满足不了使用要求，本调试方法旨在围绕ARM-GCC这一跨平台工具链进行开发，所使用到的软件及插件如下：

* segger-jlink
* arm-none-eabi-gcc toolchain
* make
* shell
* vs code editor
* vscode-cortex-debug plugin

正如以上所需的开发环境，除了ARM-GCC工具链是必须的，其他包括工程组织方法、编辑器、及调试硬件都是可选的，为了方便演示则选取上述的相关软件环境。
